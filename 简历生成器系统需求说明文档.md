# **项目需求说明书（V2.1）**

**项目名称**：日常履历沉淀与智能简历生成器\
**版本**：2.1\
**日期**：2026年6月11日\
**负责人**：陈凯鑫

## **1. 项目背景**

当前学生在求职、实习或保研过程中，普遍存在"平时不记录、写简历时遗忘成果"的问题。大量课程项目、竞赛经历、技能学习等日常积累未被及时沉淀，导致简历内容空洞、描述缺乏数据支撑。\
本项目旨在构建一个 **"记录‑分析‑生成"** 闭环系统：用户通过日常记录自然积累履历素材，系统自动提取技能关键词并提供可视化统计，最终通过低代码拖拽式编辑器与模板市场，一键生成专业、美观的
Word 简历，显著降低简历制作门槛。

## **2. 项目目标**

提供一个**轻量级日常履历记录**工具，支持用户随时记录学习、项目、竞赛、实习等经历。

基于自然语言分词技术，**自动识别并统计**用户掌握的技能关键词，辅助用户了解自身技能分布。

实现**低代码拖拽式简历编辑器**，用户可自由设计版式、调整组件位置与样式。

建设**模板市场**，提供多种预设简历模板，并支持用户分享自制模板。

支持**一键导出标准 .docx 格式简历文件**，格式与模板设计保持一致。

保证系统**易用、稳定、可扩展**，为后续增加 PDF
导出、多语言简历等功能预留接口。

## **4. 功能需求**

### **4.1 日常记录管理（核心）------ 详细版**

日常记录是简历数据的源头，也是系统智能分析的基础。该模块需要支持用户**低门槛、高效率**地沉淀日常成果，并提供灵活的**分类、标签、搜索**功能，便于后续简历组装时快速筛选。

#### **4.1.1 记录的数据模型**

每一条记录应包含以下字段（前后端统一）：

  ---------------------------------------------------------------------------------------------------------------------------------
  字段名         类型              必填   说明
  -------------- ----------------- ------ -----------------------------------------------------------------------------------------
  id             String            是     自动生成的唯一标识

  title          String            是     记录标题，如"电商秒杀系统设计"

  type           Enum              是     可选值：project、internship、competition、skill、certification、other

  startDate      Date              否     开始时间（如项目启动日期）

  endDate        Date              否     结束时间（若为空则表示"至今"）

  description    RichText          是     详细描述，支持纯文本 + 简单 Markdown（如粗体、列表、链接）。该字段是分词分析的主要来源。

  achievements   Array\<String\>   否     可量化的成果列表，例如"日活提升20%"、"GitHub 50星"。便于后续简历中突出亮点。

  skills         Array\<String\>   否     用户手动关联的技能标签（如"Java"、"Redis"）。**优先级高于自动分词**，用于修正识别错误。

  attachments    Array\<Object\>   否     附件列表，每个附件包含 url（图片/PDF链接）、name（文件名）、type（image/pdf）。

  link           String            否     项目链接、证书验证链接或 GitHub 仓库地址。

  isPublic       Boolean           否     是否允许其他用户（在模板市场中）看到该记录作为示例，默认为 false。

  createdAt      Date              是     创建时间（自动生成）

  updatedAt      Date              是     最后更新时间（自动更新）
  ---------------------------------------------------------------------------------------------------------------------------------

> 注：description 采用富文本而非纯文本，是为了让用户能够组织更清晰的履历描述（如分段、小标题）。系统在分词时会自动提取其中的纯文本部分。

#### **4.1.2 功能操作**

##### **（1）新增记录**

**入口**：侧边栏或顶部"➕ 添加记录"按钮。

**录入方式**：弹出模态框，依次填写字段。为降低用户心理负担，提供**快速模板**：

"完成一个课程项目" →
自动填充 type=project，提示用户填写标题、时间、描述。

"学完一项新技术" → 自动填充 type=skill，额外提示"是否关联证书"。

**草稿自动保存**：用户编辑中途关闭模态框时，自动保存为草稿（状态标记 draft=true），下次打开可继续编辑。

##### **（2）编辑记录**

每条记录右侧显示"编辑"图标，点击回填所有字段进行修改。

支持**批量编辑**：勾选多条记录后，可批量修改类型、时间范围或添加公共标签。**（3）删除记录**

单项删除：确认后删除该记录。

批量删除：勾选多条后一键删除。

**软删除机制**：为避免误删，删除的记录默认移入回收站，用户可在 30
天内恢复或永久删除。

##### **（4）查询与筛选**

**搜索框**：支持按标题、描述内容关键词模糊搜索。

**筛选器**：

按类型（多选，如只显示"项目"+"竞赛"）

按时间范围（最近一周 / 一个月 / 自定义区间）

按关联技能（如筛选所有包含"React"的记录）

**排序**：按更新时间、开始时间、结束时间升序/降序排列。

##### **（5）记录详情视图**

点击记录标题或"详情"按钮，进入独立的详情页面（或侧边抽屉）。

详情页以卡片样式展示完整信息，包括附件缩略图、成果列表、链接等。

提供"从本条记录生成简历片段"快捷按钮：系统自动将记录内容格式化为简历描述文本，并可直接添加到当前正在编辑的简历中（见低代码编辑器章节）。

#### **4.1.3 智能辅助功能**

**成果建议**：当用户在 description 中输入"负责了..."、"参与了..."等被动句式时，系统弹出提示："建议使用
STAR 法则（情境-任务-行动-结果）并量化成果"，并提供改写示例。

**技能自动补全**：在 skills 标签输入时，下拉提示基于用户历史记录中高频出现的技能词（从分词中提取）及预设词典。

**重复检测**：如果新增记录的标题或描述与已有记录高度相似（余弦相似度 \>
0.85），提示用户"该记录似乎与『xxx』重复，是否继续添加？"

#### **4.1.4 数据导入与导出**

**导入**：支持批量导入 CSV / JSON
文件（格式模板可下载），方便用户从旧系统迁移数据。

**导出**：用户可一键导出所有记录为 JSON 或 CSV，用于本地备份或二次加工。

#### **4.1.5 与简历生成的衔接**

在低代码编辑器中，"项目经历"等列表组件可**直接按类型过滤**用户日常记录。例如组件绑定 type=project 的所有记录，自动按照 startDate 倒序排列。

用户可在记录管理页面**为某条记录打上"隐藏"标记**，使其不出现在简历自动填充列表中，避免隐私内容暴露。

### **4.2 智能分析与技能词频统计**

后端使用 **HanLP** 对记录的"详细描述"字段进行中文分词。

过滤停用词（如"的"、"了"、"然后"），提取技术、技能类关键词。

统计技能词出现频次，以词云或柱状图形式在前端展示。

**自定义词典**：预先导入计算机常见技能词（如 React、Spring
Boot、PyTorch）、比赛名称、证书名称，提高分词准确率。

### **4.3 低代码拖拽式简历编辑器**

**画布区域**：模拟 A4 纸大小，支持缩放预览。

**组件库**：提供文本框（单行/多行）、图片框、列表（教育经历/项目经历）、分割线、页眉页脚等组件。

**拖拽操作**：用户可将组件从侧边栏拖入画布，自由移动、调整大小、删除。

**属性面板**：选中组件后可修改文字内容、字体、颜色、边距等基础样式。

**数据绑定**：组件可与用户已有的日常记录数据关联（例如"项目经历"列表组件绑定到若干条类型为"项目"的记录）。

**实时预览**：编辑过程中实时刷新简历样式。

**技术实现**：前端采用 react-grid-layout 或 react-dnd 快速搭建基础布局，后续逐步增强自定义能力。

### **4.4 模板市场**

**模板展示**：以缩略图形式展示预设模板及用户公开的模板。

**模板详情**：点击可查看大图预览、使用人数、评分。

**选用模板**：选择模板后自动在编辑器中加载其布局和数据绑定方案。

**上传/分享模板**：用户可将自己编辑的布局保存为模板，选择是否公开至市场。

**搜索与分类**：支持按行业（互联网、金融、设计）、风格（简约、现代、创意）筛选。

### **4.5 简历生成与导出（.docx）**

**导出流程**：用户点击导出 → 后端根据当前编辑器中的布局定义 +
绑定的用户数据 → 通过 **poi‑tl** 渲染 Word 模板 → 返回 .docx
文件供下载。

**核心机制**：

预设多套 Word 模板（.docx），其中使用 poi‑tl
标签（如 {{name}}、{{#projects}}\...{{/projects}}）定义动态区域。

编辑器生成的布局配置会映射到对应的 Word 模板及数据填充逻辑。

**支持内容**：动态表格（项目经历/教育经历多行插入）、图片（头像、证书截图）、列表、条件显示（例如没有实习经历时隐藏对应区块）。

**下载文件名**：简历_用户名_生成时间.docx。

### **4.6 用户认证与数据持久化**

支持邮箱/手机号注册登录（JWT 认证）。

所有用户记录、编辑器布局、收藏的模板均保存在 MongoDB 中。

用户可导出全部原始数据（JSON 格式）作为备份。

### **关于 poi‑tl 的选型确认**

经过对比 docx4j、Apache POI、前端 JS 库等方案，poi‑tl
在以下方面最适合本项目的简历生成场景：

**模板即 Word**：设计师或用户可直接在 Word
中设计版式、放置占位符（如 {{name}}、{{#items}}\...{{/items}}），无需编写代码。

**功能完备**：支持动态表格（简历中最核心的需求）、图片插入（尺寸可控）、条件隐藏/显示。

**开发效率**：学习曲线平滑，API
直观，且国内社区活跃，踩坑后容易找到解决方案。

**性能满足**：单份简历生成耗时 \< 500ms，用户侧无感知。

**资源消耗**：依赖轻量，不会给后端带来显著负担。

因此，**poi‑tl 将作为唯一文档生成实现方案**，不再引入 docx4j
或其他备选库（除非未来出现极特殊的格式需求）。

## **7. 难点与解决方案**

  ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  难点                                 解决方案
  ------------------------------------ --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  **拖拽编辑器的数据持久化**           使用 JSON 描述画布中每个组件的位置、大小、数据绑定关系，存入 MongoDB。加载时反序列化恢复布局。

  **模板与用户数据的自动匹配**         为每个模板定义一份"字段清单"（Schema），例如模板需要 projectList、educationList。系统根据字段名模糊匹配用户记录（如 projectList 自动关联类型为"项目"的记录），用户也可手动调整映射。

  **分词对专业术语识别不准**           在 HanLP 中加载自定义词典，收集常见编程语言、框架、证书名称（如 "TensorFlow"、"CSP"）。同时提供前端"手动添加技能词"的修正入口。

  **poi‑tl 处理图片路径**              后端在渲染图片前，先确保图片 URL 可访问（OSS 或本地存储返回绝对路径），poi‑tl 支持直接传入网络图片地址或本地 File 对象。

  **拖拽编辑时组件对齐与缩放**         使用 react-grid-layout 默认的网格系统（如 10px 网格），并提供"吸附对齐"辅助功能。

  **导出样式与编辑器预览不完全一致**   编辑器预览直接使用 HTML/CSS 模拟 Word 常见样式；导出时 poi‑tl 使用的 Word 模板也采用相同样式参数（字体、行距、边距），通过手工调校保证一致性。
  ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

## **9. 验收标准**

**功能完整性**

用户可完整走通：记录日常 → 查看技能统计 → 选择模板 → 拖拽调整 → 导出
.docx。

导出的 .docx
文件中，表格行数与用户选择的项目/经历条数一致，图片正常显示。

**非功能达标**

分词统计接口在 2 秒内返回结果（30 条记录以内）。

拖拽编辑器在 Chrome 最新版本上无明显卡顿或错位。

**准确率**

抽取 50 条典型用户的日常记录，人工核对技能词提取结果，准确率 ≥ 85%。

**兼容性**

导出的简历文件分别在 Word 2016 和 WPS 2019
中打开，排版与预期相比偏差不超过 2 毫米（按 A4 纸为标准）。
